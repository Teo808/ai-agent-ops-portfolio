Forward Deployed Engineer / AI Agent Operations

## Positioning Statement

I am deeply interested in Hermes Agent and practical AI agent deployment. I have been building and testing workflows around Hermes-style agents, OpenRouter and OpenAI-compatible providers, local models, MCP/tool integrations, Obsidian/Supermemory-style memory systems, browser automation, and agent automation stacks.

My strongest fit is testing, breaking, documenting, and improving real user workflows. I want to work where AI agents are deployed into actual environments, where reliability depends on setup quality, tool behavior, memory scoping, debugging, and clear operator documentation.

## Why Nous Should Call Me Back

I am applying from the operator side of AI agents: the side that notices when setup instructions are unclear, memory retrieval drifts, a tool is authenticated but unavailable, a browser workflow breaks, or a provider fallback quietly changes behavior.

That field-level attention is valuable for Hermes Agent because successful deployment depends on more than model quality. It depends on whether the agent can be installed, connected, trusted, debugged, explained, and improved in real environments.
