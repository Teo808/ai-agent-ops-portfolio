Case Study 01

I test AI agents like they have to survive real deployment.

My Hermes-style setup notes focus on provider setup, MCP/tool integrations, memory behavior, browser automation, messaging surfaces, and failure reporting.

Field Engineering Value

I look at agent setup from the deployment tester's side. I want the agent to survive real usage, not only a clean demo. That means reproducibility, clear failure reports, and small verification steps that help engineering teams move faster.

Areas I test
  • Nous Portal, OpenRouter, local model endpoints, and OpenAI-compatible APIs.
  • Project memory, personal memory, recall quality, source labels, and stale memories.
  • MCP/tool discovery, auth, transport errors, server availability, and naming clarity.
  • Browser navigation, form filling, screenshots, and failure recovery.
What I document
  • The exact failure message.
  • The command or UI path that reproduced it.
  • The fix and the verification step.
  • The lesson that should become a checklist.

Deployment Test Matrix

ProviderWrong model, bad endpoint, missing key, or fallback confusion.
ToolingMCP server unavailable, auth mismatch, or unclear tool name.
MemoryWrong project scope, stale global result, or missing source.
BrowserLogin wall, CAPTCHA, selector drift, or slow page state.

Hiring Signal

Forward deployed work rewards people who can move between user reports, tooling, infrastructure symptoms, and clear documentation. This is how I work: reproduce first, isolate the layer, verify the fix, and preserve the lesson.
