# Candidate Profile

## Target Role

Forward Deployed Engineer / AI Agent Operations

## Positioning Statement

Built and tested workflows around Hermes-style agents, OpenRouter and OpenAI-compatible providers, local models, MCP/tool integrations, Obsidian/Supermemory-style memory systems, browser automation, and agent automation stacks.

My strongest fit is testing, breaking, documenting, and improving real user workflows. The specific skill is turning broken paths into clear ones: setup notes, debugging guides, field reports that someone else can actually follow. That's what makes deployment reliable after the model is already good.

## Why Nous Should Call Me Back

I'm applying from the operator side: the side that notices when setup instructions are unclear, memory retrieval drifts, a tool is authenticated but unavailable, a browser workflow breaks, or a provider fallback quietly changes behavior, and then writes it down clearly enough that someone on the engineering team can act on it.

Successful deployment of Hermes Agent depends on more than model quality. It depends on whether the agent can be installed, connected, trusted, debugged, explained, and improved in real environments. That layer is where I want to work.

## Best Evidence

- Hermes-style setup and workflow testing (memory, tools, CLI integrations)
- Provider and local model experimentation
- MCP/tool integration troubleshooting
- Supermemory-style project and personal memory workflows
- Browser automation and job/content automation systems
- Setup guides, issue reports, and reproducible debugging paths, written clearly enough for someone else to follow

## Role Fit

I am not positioning myself as an ML researcher. I am positioning myself as the person who actually uses, tests, breaks, fixes, documents, and understands AI agents in the wild.

This fits agent deployment, field engineering, QA, support engineering, solutions engineering, and operations work around Hermes Agent.

## Operating Style

| Situation | How I help |
